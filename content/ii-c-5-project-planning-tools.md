## Project Planning in the Define Phase

During the Define phase of a DMAIC project, the team creates a project charter that serves as a high-level plan for the work. The charter often includes a rough timeline estimate, a problem statement, goals, and scope boundaries. This preliminary schedule is a basic project planning tool because it sets the sequence and time frame that guide later phases. In parallel, teams carry out risk mitigation planning. They identify potential threats to the project and develop actions to reduce the likelihood or impact of those risks. These planning activities, described in the Project Management Basics chapter of the ASQ CSSGB Handbook, give the team a clear roadmap and provide a basis for communicating the project’s direction to stakeholders and sponsors.

## Activity Network Diagram

An activity network diagram is a graphical planning tool that shows interdependencies between tasks. The AND is similar to a PERT chart and, like critical path analysis, can provide a top-level overview or detailed data depending on the project need. It highlights key tasks, the time required to complete each task, and the flow paths: serial (tasks that must be performed one after another) or parallel (tasks that can be performed simultaneously).

### Why the AND Matters in Six Sigma

The AND helps a project team structure the work early in the Define phase so that task sequences and timing are understood before data collection begins. It reveals where delays could occur, aids in resource allocation, and supports a realistic project timeline. By laying out dependencies, the team can identify opportunities to run tasks concurrently and compress the schedule.

### Steps to Construct an Activity Network Diagram

Using the elements described in the source material, a team can build an AND as follows:

1. **List the key tasks** required to complete the project. These tasks come directly from the project scope defined in the charter.
2. **Assign a time estimate** for each task. Estimates can come from historical data, expert judgment, or initial approximations.
3. **Determine dependencies** among tasks. Identify which tasks must finish before others can start (serial flow) and which tasks can be performed at the same time (parallel flow).
4. **Draw the network** by placing tasks as nodes or boxes and connecting them with arrows that represent the flow. The diagram visualizes the sequence and reveals the total project duration by summing the times along the longest serial chain.
5. **Review and refine** the diagram with the team and the project sponsor to reach agreement on the planned sequence and estimates. The AND can be revisited and updated as new information emerges during later DMAIC phases.

This procedure derives directly from the source description: key tasks, time to accomplish the tasks, and serial or parallel flow paths.

## Integrating Planning Tools with the Project Charter

The project charter supplies a rough timeline estimate, while the AND provides a detailed breakdown of tasks and their logical order. Together they constitute the core of the project plan. Risk mitigation actions can be layered onto the AND by noting contingency plans for critical tasks. During the Define tollgate review, the champion or leadership board evaluates the plan, which includes the charter and any network diagrams, before the team proceeds to the Measure phase.

## Exam Tips

1. **Distinguish the AND from other diagrams:** The CSSGB exam may ask you to identify that an activity network diagram shows task dependencies and duration. Do not confuse it with flowcharts or SIPOC diagrams, which model process steps and supplier-customer relationships.
2. **Interpret serial vs. parallel paths:** Given a simple AND, you may need to identify which tasks can run concurrently. Parallel tasks reduce total project time, while serial tasks determine the minimum completion time.
3. **Connect AND to PERT and critical path analysis:** Recognize that the AND is a project planning tool analogous to a PERT chart. A scenario that requires a top-level project schedule should point you toward an AND or PERT chart.
4. **Know the planning role of the charter:** The charter includes a rough timeline estimate, making it the earliest project planning document. An exam question may ask you to identify where a project schedule first appears, the correct answer is the project charter.
5. **Apply risk mitigation within planning:** Risk identification and mitigation planning occur during the Define phase as part of the overall project plan. You may be asked to link contingency actions to specific tasks on a project schedule.
